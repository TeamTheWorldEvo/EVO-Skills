# Step 4：安全配置 — 沙箱与工作区隔离

这是 settings.json 中的硬配置，不是行为建议。

## 4.1 Bash 沙箱

**解释：**

> **配置：Bash 沙箱**
>
> **它做什么**：所有通过 Bash 工具执行的 shell 命令都在隔离环境中运行。沙箱限制了命令能访问的文件系统范围。
>
> **`sandbox.enabled: true`** → 命令在沙箱中运行，无法访问沙箱外的文件。
> **`sandbox.enabled: false`** → 命令以正常权限运行，可以访问系统中的任何文件。
>
> **`autoAllowBashIfSandboxed: true`**（配套选项）→ 沙箱开启时，Bash 命令自动跳过权限询问。因为沙箱本身就是安全保证，再加询问就烦人了。
>
> **推荐**：开启。这是最底层的安全保障。
>

> - **B) 只开启沙箱** — 安全但每次 Bash 都要确认
> - **C) 不开** — 我相信这个项目的执行环境

## 4.2 额外工作目录

**解释：**

> **配置：额外工作目录（additionalDirectories）**
>
> **它做什么**：控制 Claude Code 可以访问哪些**工作区以外的目录**。
>
> **`[]`（空数组）** → 只允许当前项目目录及子目录。
> **`["/some/path"]`** → 额外允许指定的目录。比如你的项目依赖一个共享库在别的路径，可以加进来。
>
> **和工作目录约束规则的关系**：规则是 AI 自觉，这个是系统强制。两者配合：规则让 AI 不往那想，配置让它想去也去不了。
